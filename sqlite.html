Инструкция по использованию SQLite Browser

1. Создание и модификация структуры (DDL)

Вы можете управлять таблицами прямо в интерфейсе через вкладку «Структура»:

  • Создание таблицы: Нажмите кнопку «+ Создать таблицу» наверху вкладки «Структура». Задайте имя и добавьте колонки, указав их типы (TEXT, INTEGER, REAL, BLOB), а также параметры Primary Key (PK) и Not Null (NN).
  • Добавление новой колонки: Раскройте схему нужной таблицы и нажмите кнопку «+ Добавить колонку».
  • Удаление таблицы: Нажмите «Удалить таблицу» внутри параметров этой таблицы в схеме.

2. Добавление и изменение содержимого (DML)

Во вкладке «Просмотр данных» вы можете управлять записями:

  • Добавление записей: Нажмите кнопку «+ Добавить строку» в панели инструментов. Программа построит форму, соответствующую структуре таблицы.
  • Редактирование записей: Если в таблице есть поддержка уникальных идентификаторов (rowid), справа от каждой строки отобразится кнопка редактирования (карандаш). Вы можете редактировать любые поля, а для BLOB-колонок — загружать новые файлы.
  • Удаление записей: Нажмите кнопку корзины в строке действия для удаления записи.

3. Выполнение произвольных SQL-запросов

Вкладка «SQL-запрос» поддерживает любые команды SQLite:

  • Вы можете выполнять запросы SELECT, INSERT, UPDATE, DELETE, ALTER TABLE, CREATE TABLE и т.д.
  • Для быстрой вставки воспользуйтесь выпадающим списком «Шаблоны запросов».
  • Запуск запроса осуществляется кнопкой «Выполнить» либо сочетанием клавиш Ctrl + Enter (Cmd + Enter на Mac).

4. Генератор баз данных из файлов

Вкладка «Генератор БД» позволяет быстро упаковать файлы и папки в локальную базу SQLite:

  • 1 файл — 1 запись: Каждый файл запишется в отдельную строку таблицы files в оригинальном или сжатом (ZIP) виде.
  • 1 подпапка — 1 запись: Группирует файлы по папкам верхнего уровня и создает ZIP-архив для каждой папки, сохраняя относительные пути файлов внутри архива.
  • Скачивание: После генерации вы можете просмотреть и протестировать получившуюся структуру, а затем скачать готовый файл generated_archive.db.
